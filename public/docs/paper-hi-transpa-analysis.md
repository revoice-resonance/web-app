# 论文分析：HI-TransPA（听损个人助手）

**论文**：HI-TransPA: Hearing Impairments Translation Personal Assistant  
**作者**：Shi Feng, Xuancheng Huang, Shaohan Huang 等  
**发表**：arXiv:2511.09915, 2025  
**聚焦**：多模态融合、质量评分拒绝采样、课程学习策略在小模型上的应用

---

## 1. 核心贡献

| 贡献 | 内容 |
|------|------|
| **系统** | 融合音频+唇动的多模态听损翻译助手，基于 Qwen2.5-Omni-3B 小模型 |
| **质量评分系统** | 复合评分 = 0.6×ASR置信度 + 0.4×唇部识别置信度，自动过滤低质量样本 |
| **课程学习策略** | 两阶段训练：先学高质量接受样本(3 epoch) → 再学困难拒绝样本(5 epoch) |
| **性能** | CER = 0.27，优于 Whisper-large-V3 (0.32) 和 Qwen2.5-Omni 7B (0.42) |

## 2. 实验方法论详解（可迁移部分）

### 2.1 多模态特征融合架构

#### 音频通路
```
输入音频 (16 kHz, 单声道)
  ├─ 预处理：MFCC + Mel频谱
  ├─ 编码器：Whisper 音频编码器
  └─ 输出：音频特征向量
```

#### 唇部视觉通路（⭐ 新颖设计）
```
输入视频帧
  ├─ 唇部检测：468 个面部关键点（MediaPipe）
  ├─ 稳定化裁剪：插值处理 → 归一化裁剪窗口
  ├─ 编码器：SigLIP ViT (Vision Transformer) 提取视觉特征
  ├─ 空间-时间压缩：3D-Resampler (64 可学习查询)
  └─ 输出：压缩的视觉特征向量 (shape: 64 × hidden_dim)
```

**迁移建议**：
- 3D-Resampler 可直接应用于我们的多模态系统
- 若仅用音频，可简化为一维特征重采样
- MediaPipe 唇部检测成熟度高，可用于唇读辅助评估

#### 多模态融合
```
音频特征 + 唇部特征
  ↓
Qwen2.5-Omni-3B 多模态编码器
  ↓
指令遵循翻译生成
```

**关键参数**：
- 融合点：LLM 输入层（不在编码器层面融合，而是在语言模型层）
- 优势：充分利用 LLM 的跨模态推理能力

### 2.2 质量评分与拒绝采样（⭐⭐ 高度可迁移）

#### 复合质量评分公式

$$S_{comp} = 0.6 \times S_{audio} + 0.4 \times S_{video}$$

其中：
- **$S_{audio}$** = ASR 模型对文本转录的置信度（softmax 最大值）
- **$S_{video}$** = 唇部关键点检测置信度（MediaPipe 输出的置信度均值）

#### 采样策略
```
所有录音样本
  ├─ 计算复合评分
  ├─ 阈值划分（通常取 0.5~0.7）
  │
  ├─ 分类 1: "Accepted" (高质量)
  │   └─ 样本评分 > threshold
  │
  └─ 分类 2: "Rejected" (低质量)
      └─ 样本评分 ≤ threshold
```

**迁移到 Project Resonance**：
- 用 **StepFun ASR 置信度** 替代 Whisper
- 用**我们现有的唇读模型** (若有) 或 MediaPipe 替代 SigLIP
- 设定 threshold = 0.6 作为初始值，可通过消融调整
- 自动标记低质量录音，供人工复查或重新采集

### 2.3 课程学习策略（⭐⭐ 核心训练方法）

#### 两阶段课程学习

```
阶段 1: 学习阶段 (Learning Phase) - 3 epochs
  ├─ 数据集：Accepted 样本（高质量）
  ├─ 目的：快速学习可靠的模式
  ├─ 学习率：基础学习率 (e.g., 2e-5)
  └─ 输出：收敛到局部最优

↓ (经过 3 个 epoch，模型已初步学习)

阶段 2: 强化阶段 (Hardening Phase) - 5 epochs
  ├─ 数据集：Rejected 样本（低质量、噪声、困难样本）
  ├─ 目的：增强鲁棒性，学习边界情况
  ├─ 学习率：降低 0.5~0.8 倍（防止灾难遗忘）
  └─ 输出：模型对困难样本的容错能力提升
```

**关键数据**（论文实验结果）：

| 训练策略 | CER | 性能提升 |
|--------|-----|---------|
| 直接训练（所有样本混合） | 0.32 | — |
| 仅 Accepted 样本 | 0.29 | —7% |
| Accepted → Rejected（两阶段课程学习） | **0.27** | **15%** |

**迁移建议**：
- 可用于我们的 SIC → SID 过渡阶段
- 在 SID 阶段先用健康对照组数据(3 epoch)，再用构音障碍数据(5 epoch)
- 类似的思路可应用于 SDD 个人注册微调

### 2.4 关键超参数

| 参数 | 值 | 说明 |
|------|-----|------|
| 基础模型 | Qwen2.5-Omni-3B | 轻量级多模态 LLM |
| 质量评分阈值 | 0.6 | ASR+唇部置信度综合阈值 |
| 高质量数据 epochs | 3 | 学习阶段 |
| 低质量数据 epochs | 5 | 强化阶段 |
| 学习率衰减 | 0.5~0.8× | 阶段 2 的学习率缩放 |
| 音频采样率 | 16 kHz | 与标准一致 |
| 唇部关键点数 | 468 | MediaPipe Facemesh 标准 |

### 3.1 三阶段训练的对应关系

| Resonance 阶段 | 论文对标 | 融合点 |
|--------------|---------|-------|
| SIC（健康对照基线） | 论文的"学习阶段"概念 | 都使用高质量数据快速建立基线 |
| SID（构音障碍通用） | 论文的"强化阶段"概念 | 都在基线上加入困难样本，增强泛化性 |
| SDD（个人定制） | 论文的"个人适配"概念 | 都使用少量个人数据微调模型 |

**关键启示**：论文将"数据质量分层"与"阶段式训练"相结合，而 Resonance 已有"三阶段"框架。**最优策略是两者结合**：
- 在 SIC 阶段使用质量评分筛选最干净的健康语音
- 在 SID 阶段加入 Accepted → Rejected 的课程学习
- 在 SDD 阶段用个人注册数据进行高度定制化微调

## 4. 关键发现与启示

### 4.1 质量评分的核心价值

**发现**：高质量数据与低质量数据混合训练反而劣于分阶段学习

$$\text{All Samples} \text{ (CER 0.32)} > \text{Only Accepted} \text{ (CER 0.29)} > \text{Curriculum} \text{ (CER 0.27)}$$

**启示**：
- 盲目扩大数据集反而伤害性能（标签噪声、特征混杂）
- **数据质量 > 数据量**，这对数据稀缺的病理语音领域尤为重要
- 我们应在 SIC 和 SID 阶段都实施质量评分，过滤掉低于 threshold 的录音

### 4.2 小模型 + 大策略

**发现**：3B 参数模型通过多模态融合 + 课程学习，性能逼近 7B 模型

| 模型 | 参数量 | CER |
|------|--------|-----|
| Whisper-large-V3 | 1550M | 0.32 |
| Qwen2.5-Omni 7B | 7B | 0.42 |
| **Qwen2.5-Omni 3B + 课程学习** | **3B** | **0.27** ⭐ |

**启示**：
- 算法优化可显著弥补参数量的劣势
- 对 Project Resonance 的启发：不必追求最大的 ASR 模型，而应优化训练策略
- 质量评分 + 课程学习的组合效果是参数量 2.3 倍的模型都无法达到的

### 4.3 多模态信息的互补性

**发现**：ASR 置信度 + 唇部置信度的线性组合（0.6:0.4）优于单模态评分

**启示**：
- 在多源数据采集场景下，应设计**复合评分机制**
- 对我们的场景扩展：可考虑 ASR 置信度 + 音质评分 + 时长完整性的三维评分
- 权重可通过小规模验证集进行调优

## 6. 质量评分的快速落地指南

### 6.1 最小可行实现（MVP）

```python
# 伪代码：计算每条录音的质量评分
def compute_quality_score(audio_path: str) -> dict:
    # 步骤 1: 用 StepFun ASR 转录并获取置信度
    transcript, asr_confidence = stepfun_asr(audio_path)
    
    # 步骤 2: 计算音质评分（SNR / 静音率 / 频率分布）
    quality_score = compute_audio_quality(audio_path)
    
    # 步骤 3: 复合评分
    S_comp = 0.6 * asr_confidence + 0.4 * quality_score
    
    return {
        'file': audio_path,
        'asr_confidence': asr_confidence,
        'audio_quality': quality_score,
        'composite_score': S_comp,
        'category': 'Accepted' if S_comp > 0.6 else 'Rejected'
    }

# 步骤 4: 批量处理，生成质量评分报告
results = [compute_quality_score(f) for f in all_recordings]
```

### 6.2 阈值参考

| 场景 | 推荐阈值 | 说明 |
|------|---------|------|
| SIC（健康语音基线） | **0.7** | 要求最严格，只用最干净的健康样本 |
| SID（构音障碍通用） | **0.6** | 中等严格，允许部分噪声但保留信息量 |
| SDD（个人定制） | **0.5** | 最宽松，个人数据有限，尽量保留 |

### 6.3 与现有评测流程的融合

```
现有流程：采集 → 人工标注 → 训练
          ↓
新增步骤：采集 → 质量评分 → 分类处理 → 人工标注 → 训练
                   ├─ Accepted → 直接入库
                   └─ Rejected → 标记为"低质" + 待复查
```
